## Learning Value from the Events

The three events helped me gain knowledge beyond theoretical learning. The key values I learned include:

- Effective AI systems require clear context, relevant data, and controlled output through schemas or validation.
- Cloud is not only a place to deploy applications; it also involves security, operations, cost control, monitoring, and scalability.
- Technologies such as Docker, WebSocket, WAF, GraphRAG, DevOps Agent, and MCP can solve different real-world problems.
- In team projects, communication, task assignment, and progress tracking are as important as technical skills.
- For systems that handle user data such as CVs, job data, or authentication information, access control, logging, and security should be considered from the design stage.

## Relation to My Learning Process and Project

These events helped me connect practical lessons to the AI Job Matching Platform project. Topics such as AI context, schema validation, data security, CloudFront, WAF, DevOps Agent, Docker, and teamwork can all influence how the system is designed, implemented, and operated.

From these sessions, I realized that a good cloud system should not only work technically. It should also be maintainable, observable, secure, and ready for future extension.
