### Week 1 Objectives:

* Become familiar with the learning program structure, progress tracking method, and workshop report requirements.
* Understand the role of AWS Identity and Access Management (IAM) in identity and permission management on AWS.
* Distinguish between basic IAM components such as IAM User, IAM Group, IAM Role, and IAM Policy.
* Learn the Least Privilege principle to grant only the minimum permissions required for users or services.
* Learn how AWS Budget helps monitor AWS costs and reduce the risk of unexpected charges.
* Understand why the root account should not be used for regular learning or deployment activities.

### Week 1 Achievements:

* **Understood the role of IAM in AWS:** Learned that IAM is a core service for managing identities, permissions, and access control for AWS resources.
* **Distinguished between basic IAM components:** Understood the differences between IAM User, IAM Group, IAM Role, and IAM Policy, and when each component should be used.
* **Learned the Least Privilege principle:** Understood that permissions should not be broader than necessary because excessive permissions can introduce security risks.
* **Recognized the risks of using the root account:** Learned that the root account should only be used for special account-level tasks, not for regular learning or deployment activities.
* **Understood how AWS Budget supports cost control:** Learned how Budget helps monitor spending, configure budget alerts, and reduce the risk of unexpected charges.
* **Built a foundation for the following weeks:** Established basic security and cost management knowledge before moving on to networking, EC2, serverless, and other AWS services.
