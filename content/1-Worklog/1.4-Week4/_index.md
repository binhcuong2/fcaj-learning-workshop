### Week 4 Objectives:

* Learn the serverless model on AWS.
* Understand the roles of AWS Lambda, Amazon API Gateway, Amazon DynamoDB, and Amazon CloudWatch Logs.
* Learn how Lambda receives an event, processes business logic, and returns a response.
* Understand API Gateway as the HTTP entry point for frontend requests.
* Study DynamoDB basics including Table, Item, Partition Key, Sort Key, Query, and Scan.
* Practice or observe Lambda testing with simulated JSON events.
* Learn how CloudWatch Logs helps inspect Lambda errors.

### Week 4 Achievements:

* **Understood serverless benefits:** Learned that serverless reduces direct server management work.
* **Understood Lambda usage:** Learned that Lambda is used to process backend logic from event input.
* **Understood API Gateway:** Learned that API Gateway receives frontend HTTP requests and forwards them to Lambda.
* **Understood DynamoDB basics:** Learned that DynamoDB is a NoSQL database suitable for key-based access patterns.
* **Learned Lambda testing workflow:** Understood how to test Lambda with JSON events before connecting it to a frontend.
